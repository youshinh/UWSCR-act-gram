# **Go \+ Wails 開発環境構築ガイド (Windows環境向け最短ルート)**

本システム（AIネイティブRPAラッパー）の開発に必要な環境を、最短手順で構築します。Wailsはバックエンドを「Go」、フロントエンド（UI）を「Web技術（HTML/JS等）」で構築するフレームワークです。

## **Step 1: 必須ソフトウェアの一括インストール**

フロントエンドのビルドには Node.js が、バックエンドのコンパイルには Go が必要です。

Windows PowerShell を**管理者権限**で開き、以下のコマンドを順番に実行してください。

\# 1\. Node.js のインストール (LTS版)  
winget install OpenJS.NodeJS.LTS

\# 2\. Go 言語のインストール  
winget install GoLang.Go

\# 3\. Visual Studio Code のインストール (※すでに入っている場合は不要)  
winget install Microsoft.VisualStudioCode

**注意:** インストールが完了したら、環境変数（PATH）を反映させるため、**一度開いているPowerShell（またはコマンドプロンプトやVS Code）をすべて閉じ、新しく開き直してください。**

## **Step 2: インストールの確認**

新しく開いたPowerShellで、以下のコマンドを打ち、バージョン番号が表示されればStep 1は成功です。

go version  
\# 例: go version go1.22.x windows/amd64

node \-v  
\# 例: v20.x.x

npm \-v  
\# 例: 10.x.x

## **Step 3: Wails CLI のインストールと環境診断**

次に、Wailsプロジェクトを作成・ビルドするためのコマンドラインツール（CLI）を、Goのパッケージマネージャー経由でインストールします。

\# Wails CLI をインストール  
go install \[github.com/wailsapp/wails/v2/cmd/wails@latest\](https://github.com/wailsapp/wails/v2/cmd/wails@latest)

インストールが終わったら、Wailsの開発環境が整っているか自動診断するコマンドを実行します。

wails doctor

出力画面で SUCCESS や緑色のチェックマークが多くついていれば準備完了です。（WebView2 Runtime が無いと警告が出た場合は、Windowsアップデート経由か、画面の指示に従ってインストールしてください。Windows 11なら標準で入っています）。

## **Step 4: プロジェクトの初期化（ひな形の作成）**

開発用のフォルダ（例: Documents\\Dev など）に移動し、プロジェクトのひな形を作成します。

今回は、フロントエンドの記述がシンプルで軽量な **Svelte（スベルト）** というフレームワークをベースにしたテンプレートを使用します。

\# ai\_rpa\_agent という名前のプロジェクトを作成  
wails init \-n ai\_rpa\_agent \-t svelte

\# 作成されたフォルダに移動  
cd ai\_rpa\_agent

## **Step 5: 開発モードでの起動（動作確認）**

いよいよ起動です。以下のコマンドを実行すると、初回はフロントエンドのパッケージ（npmモジュール）のダウンロードとコンパイルが走ります。

wails dev

数分待つと、独立した美しいウィンドウ（デスクトップアプリ）が立ち上がります。

* **Wails Devモードの特徴:** 起動したまま、VS Code等でコード（Go側でもUI側でも）を書き換えて保存すると、自動的に画面がリロード（ホットリロード）されます。これにより、開発スピードが劇的に上がります。
