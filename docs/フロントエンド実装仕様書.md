# **UWSCR::act-gram フロントエンド実装仕様書 (Wails \+ Svelte)**

## **1\. 概要**

本モジュールは、UWSCRの拡張エージェント「UWSCR::act-gram」のコントロールセンター（GUI）を提供する。

Wailsフレームワークを使用し、フロントエンドは Svelte で構築する。AI（ローカルLLM）はこの仕様に基づき、.svelte コンポーネントおよび関連するフロントエンドロジックを実装すること。

## **2\. コア機能要件**

1. **APIキーの入力と保存**  
   * Gemini, Anthropic 等のAPIキーを入力するフォーム。  
   * セキュリティ観点から、入力されたキーはフロントエンドには保持せず、Wailsのバインディング経由でGoのセキュア関数（SaveAPIKey等）へ渡し、OSのCredential Managerへ格納させる。  
2. **動的モデル検出 (Dynamic Discovery) と設定**  
   * 「Brain層（思考）」「Eye層（視覚）」「Utility層（反射）」の3つのレイヤーに対して、使用するLLMモデルをプルダウンで選択させる。  
   * プルダウンのリストは静的ではなく、Go側の関数（FetchAvailableModels等）を呼び出して動的に生成する。  
   * Svelteの \#await ブロックを活用し、取得中のローディング状態をエレガントに実装すること。

## **3\. 推奨コンポーネント構成**

* App.svelte: 全体のレイアウトと状態管理の親コンポーネント。  
* components/CredentialInput.svelte: APIキー入力用コンポーネント（パスワードマスク必須）。  
* components/LayerConfig.svelte: Brain, Eye, Utility 各層のモデルを選択するカード型コンポーネント。

## **4\. Wailsバインディング連携イメージ (JavaScript側)**

ローカルLLMへの指示：Go側で定義される以下のメソッドを window.go.main.App から呼び出す前提でUIを構築すること。

* SaveConfig(layer string, provider string, model string)  
* FetchModels(provider string) \-\> Promise\<string\[\]\>
